# include: ဖိုင်ကို လှမ်းခေါ်မယ်။ 
ဖိုင်မရှိရင် Warning (သတိပေးချက်) ပဲ ပြပြီး၊ အောက်က code တွေ ဆက်အလုပ်လုပ်မယ်။

# require: ဖိုင်ကို လှမ်းခေါ်မယ်။ 
ဖိုင်မရှိရင် Fatal Error ပြပြီး၊ အဲ့ဒီနေရာမှာတင် System တစ်ခုလုံး အလုပ်ရပ်သွားမယ်။

# include_once / require_once: 
နောက်က _once ပါသွားရင် PHP က ဒီဖိုင်ကို အရင်က ခေါ်ထားပြီးသားလား ဆိုတာ စစ်ပေးတယ်။ 
ခေါ်ထားပြီးသားဆို ထပ်မခေါ်တော့ဘူး။ (Error တွေ၊ Function name တူနေတဲ့ ပြဿနာတွေကို ကာကွယ်ဖို့)။

===================================

ဘာတွေ ရှောင်ရမလဲ (What to skip/avoid)

နေရာတကာ include သုံးတာ ရှောင်ပါ: 
အရေးကြီးတဲ့ File တွေ (ဥပမာ - Database connection, Core functions) တွေကို require_once ပဲ သုံးရမယ်။ 
include သုံးထားလို့ Database မချိတ်မိဘဲ အောက်က Code တွေ ဆက် run နေရင် Security ပိုင်းမှာရော၊ Data ပိုင်းမှာပါ ပြဿနာတက်နိုင်တယ်။

Spaghetti Code: File တွေကို တစ်ခုထဲမှာ တစ်ခု၊ အဆင့်ဆင့် လှမ်းခေါ်တာမျိုး (Nested includes) မလုပ်ပါနဲ့။ 
ဘယ် File ကနေ ဘာလာနေမှန်း မသိရတော့ဘဲ ရှုပ်ယှက်ခတ်သွားမယ်။

Laravel ရောက်ရင်: Framework တွေ သုံးလာတဲ့အခါ Composer ရဲ့ autoload ဆိုတဲ့ စနစ်က file တွေကို auto ခေါ်ပေးသွားမှာမို့လို့၊ 
manual ရေးရတာ နည်းသွားလိမ့်မယ်။ 
ဒါပေမယ့် Core PHP သဘောတရားကို နားလည်ထားဖို့က မရှိမဖြစ် လိုအပ်တယ်။
